Procedimiento Detallado

1. Instalación y Configuración de OpenSSH Server

Instalación del paquete:

sudo apt update
sudo apt install openssh-server -y

Verificación del servicio:

sudo systemctl status ssh

Configuración básica en /etc/ssh/sshd_config:

# Puerto por defecto (puede cambiarse para mayor seguridad)
Port 22

# Deshabilitar login directo de root
PermitRootLogin no

# Permitir autenticación por contraseña temporalmente
PasswordAuthentication yes

Reinicio del servicio:

sudo systemctl restart ssh

2. Autenticación con Llaves SSH

Generación de par de llaves en el cliente:

ssh-keygen -t rsa -b 4096 -C "esaudreyes10@gmail.com"
Resultado: Se generaron llaves RSA de 4096 bits guardadas en /root/.ssh/id_rsa (privada) y /root/.ssh/id_rsa.pub (pública).

Copia de la llave pública al servidor:

ssh-copy-id rockero@192.168.0.13
Resultado: 1 llave agregada exitosamente al servidor. Autenticación configurada para el usuario "rockero".

Desactivar autenticación por contraseña (hardening):

# Editar /etc/ssh/sshd_config
PasswordAuthentication no

# Reiniciar servicio para aplicar cambios
sudo systemctl restart ssh

3. Protección con Fail2Ban

Instalación de Fail2Ban:

sudo apt install fail2ban -y

Configuración básica:

# Crear archivo de configuración local
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local
sudo nano /etc/fail2ban/jail.local

Configuración de la sección SSH:

[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 600
Explicación:
- maxretry = 3: 3 intentos fallidos antes de bloquear
- bantime = 600: Bloqueo de 10 minutos (600 segundos)

Activación del servicio:

sudo systemctl enable fail2ban
sudo systemctl start fail2ban

Verificación del estado:

sudo fail2ban-client status sshd

Resolución de Problemas Comunes

Problema Solución Comando de Verificación
No se puede conectar por SSH Verificar que el servicio esté activo y el firewall permita el puerto sudo systemctl status ssh
sudo ufw status
Autenticación falla con llaves Verificar permisos de archivos ~/.ssh (600 para privada, 644 para pública) ls -la ~/.ssh/
ssh -v usuario@servidor
Fail2Ban no bloquea intentos Verificar configuración y logs de Fail2Ban sudo fail2ban-client status sshd
sudo tail -f /var/log/fail2ban.log
IP bloqueada incorrectamente Desbloquear IP manualmente sudo fail2ban-client set sshd unbanip IP_ADDRESS
