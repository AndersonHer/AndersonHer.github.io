Laboratorio CCNA Avanzado – Mega Data-Center

Documentación completa del laboratorio con EtherChannel, VTP, RSTP, Seguridad L2/L3, VPN y AAA

1) Objetivo

  • Implementar un data-center jerárquico con topología Core-Distribution-Access.
  • Segmentar red mediante VLANs (10-14) y troncales EtherChannel LACP.
  • Proteger capa 2: RSTP, DHCP Snooping, DAI, Port-Security.
  • Configurar servicios de infraestructura: DHCP, DNS, NTP, SYSLOG, AAA (TACACS+/RADIUS).
  • Establecer túnel VPN IPsec sitio-a-sitio entre ROUTERDATACENTER y RB.
  • Controlar tráfico inter-VLAN mediante ACLs extendidas.

2) Topología y Equipamiento

  • Core: 2× Multilayer 3560 (SWP1-SWP2)
  • Distribution: 4× 3560 (SWD1-SWD4)
  • Access: 8× 2960 (SWA-SWH)
  • Servidores: DHCP/DNS/HTTP (192.168.140.10), SYSLOG (192.168.140.11), NTP (192.168.140.12), AAA (192.168.140.11)
  • Routers: ROUTERDATACENTER (HQ) ↔ RB (Remote Branch) vía VPN

5.6 ACLs Extendidas – Control Inter-VLAN

! VLAN10 → Otras
ip access-list extended VLAN10_TO_OTHERS
 permit icmp 192.168.10.0 0.0.0.255 any
 permit udp 192.168.10.0 0.0.0.255 any eq 53
 permit tcp 192.168.10.0 0.0.0.255 192.168.13.0 0.0.0.255 eq 445
 deny ip 192.168.10.0 0.0.0.255 192.168.11.0 0.0.0.255
 deny ip 192.168.10.0 0.0.0.255 192.168.12.0 0.0.0.255

5.9 VPN IPsec – ROUTERDATACENTER ↔ RB

! ROUTERDATACENTER
access-list 110 permit ip 192.168.140.0 0.0.0.255 192.168.120.0 0.0.0.255
access-list 110 permit ip 192.168.10.0 0.0.0.255 192.168.120.0 0.0.0.255
crypto isakmp policy 10
 encryption aes 256
 authentication pre-share
 group 5
crypto isakmp key vpnpa66 address 10.0.3.2
crypto ipsec transform-set VPN-SET esp-aes esp-sha-hmac
crypto map VPN-MAP 10 ipsec-isakmp
 set peer 10.0.3.2
 set transform-set VPN-SET
 match address 110
interface GigabitEthernet0/0/0
 crypto map VPN-MAP

! RB (config idéntica con peer 10.0.0.1)
