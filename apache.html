LABORATORIO: SERVIDOR WEB APACHE

Configuración de Virtual Hosts, Logs Separados y Rotación de Logs

Objetivo del Laboratorio

Configurar un servidor web Apache que sirva dos sitios distintos utilizando Virtual Hosts, registrar sus logs de manera separada y asegurar la rotación automática de los mismos para mantener un entorno profesional y seguro.

Procedimiento Implementado

1. Instalación de Apache

Instalación y verificación del servicio Apache en el sistema.

sudo apt update
sudo apt install apache2 -y
sudo systemctl enable apache2
sudo systemctl start apache2

2. Creación de Directorios y Páginas HTML

Configuración de directorios para cada sitio web y creación de páginas HTML básicas.

sudo mkdir -p /var/www/sitio1.com
sudo mkdir -p /var/www/sitio2.com
sudo chown -R www-data:www-data /var/www/sitio1.com /var/www/sitio2.com

3. Configuración de Virtual Hosts

Creación de archivos de configuración para cada sitio con rutas de logs separadas.

# Archivo: /etc/apache2/sites-available/sitio1.com.conf
<VirtualHost *:80>
    ServerName sitio1.com
    DocumentRoot /var/www/sitio1.com
    ErrorLog /var/log/apache2/sitio1.com/error.log
    CustomLog /var/log/apache2/sitio1.com/access.log combined
</VirtualHost>

4. Configuración de Logs Separados

Creación de directorios específicos para logs de cada sitio con permisos adecuados.

sudo mkdir -p /var/log/apache2/sitio1.com
sudo mkdir -p /var/log/apache2/sitio2.com
sudo chown -R www-data:adm /var/log/apache2/sitio1.com /var/log/apache2/sitio2.com

5. Rotación de Logs con Logrotate

Configuración de rotación automática de logs para mantener el sistema limpio y seguro.

# Archivo: /etc/logrotate.d/sitio1.com
/var/log/apache2/sitio1.com/*.log {
    daily
    rotate 7
    missingok
    notifempty
    compress
    delaycompress
    create 640 www-data adm
    sharedscripts
    postrotate
        /usr/sbin/apache2ctl graceful > /dev/null
    endscript
}
